Register row R-20. Carved out of the fork-C ratification (decision paper
2026-07-07, ratified at anchor aff2f8ac): the decision grew from a plain
string-kind retype into one atomic full-stack change that cannot be split across
item-0076 (contract fields) and item-0081 (verify chain) without a half-typed
intermediate that will not compile, so it owns R-20 alone. Both 0076 and 0081
now consume the already-typed forms rather than deriving them.

The ratified shape: endpoint.#URL is the single canonical URL form for strike
(RFC URL, https mandatory, host mandatory, query and fragment forbidden);
endpoint.#URL is not borrowed as-is but repaired first, and ParseURL rejection of
query/fragment moves to the parse/schema boundary on both surfaces. issuer is
#URL on the wire and endpoint.Address in Go at all three fields (#OIDCConfig,
#SBOMSigner, sealed #ProvenanceOIDC); Address gains MarshalJSON/UnmarshalJSON/
String, all projecting through URL(). Wire-neutral because Address.MarshalJSON
emits the identical https string today's issuer: string field emits, so sealed
provenance bytes are byte-identical and golden regeneration is not required; the
round-trip property test plus an empty golden byte-diff falsify wire-neutrality
before commit. identity is primitive.#StringOrURI (RFC 7519: a string; a value
containing a colon must be a URI; compared case-sensitive verbatim) with
@go(Identity), staying a Go string; #URL is rejected for identity because it
would drop every email signer and every non-https URI (SPIFFE). Grammar
enforcement for the colon-implies-URI rule is Go-side in the validate-after-parse
phase; the CUE side is the documented scalar. primitive.#StringOrURI is available
for reuse by other JWT StringOrURI fields (aud, #OIDCConfig.audience), out of
scope here.

Every CUE change is individually confirmation-gated per AGENTS.md. The
byte-exact field set, file list, and grammar regex are authored at execution
time against the then-current pin, not stored here.
