Resolved to option B (dogfooded strike lane), 2026-07-11: the gate is a strike
lane doing the work with the forge as status reporter only, not a minimal
GitHub Actions run of `make check` verbatim. Option A was the fast path but it
cements the Makefile as the CI entry point; B is the posture-consistent path
AND the orchestration substrate that lets the whole build-toolchain arc retire
the Makefile (item-0096 deletes the file against this lane). Choosing B here is
what makes full Makefile elimination a beta prerequisite rather than a
post-beta upgrade.

Two grounded dependencies carry over: item-0016 presupposes a CI to fail
("fails CI rather than relying on review"), so this item is its substrate and
should land before or with it; and the raw coverage total includes generated
code that the 100 percent target in docs/DEVELOPMENT.md excludes, so the
ratchet pins the raw metric for simplicity, and any target-versus-actual
comparison needs a separate generated-code-excluded measurement.

Sequencing: leads the build-toolchain Makefile-elimination chain -- the lane
must exist before item-0096 can delete the aggregate `make check` into it.
