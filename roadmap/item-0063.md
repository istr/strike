Open design fork, to be ratified before any instruction is authored:
(A) minimal GitHub Actions -- checkout plus a pinned Go toolchain, no
third-party actions -- versus (B) a dogfooded strike lane doing the work with
the forge as status reporter only. A is the fast path; B is the
posture-consistent path and a natural post-beta upgrade even if A ships first.
Two grounded dependencies: item-0016 presupposes a CI to fail ("fails CI
rather than relying on review"), so this item is its substrate and should land
before or with it; and the raw coverage total includes generated code that the
100 percent target in docs/DEVELOPMENT.md excludes, so the ratchet pins the
raw metric for simplicity, and any target-versus-actual comparison needs a
separate generated-code-excluded measurement.
