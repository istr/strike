Drift-invariant notes:

Origin: during the ADR-048 concept-tier / endpoint @go work, a field named
after a wire noun (e.g. Host) was found carrying a concept type (e.g.
endpoint.Address) and read as x.Host.Host() -- semantically the Address, not a
Host. The stuttering accessor x.<Noun>.<Noun>() is the observable tell of an
incomplete @go mapping: the concept side should carry the concept name. This is
a reification candidate in the Phase-1 sense -- a human-caught violation
promoted to a permanent gate.

Why typed, not grep (ratified direction): the discriminator is that the
receiver's named type name differs from the trailing selector. A go/types
predicate encodes that exactly and has effectively no false positives: a value
legitimately typed after its name (url.URL() where the field is a URL) does not
trip, and no allowlist is needed. A backreference grep (.X.X()) is the cheaper
ship-now option but matches comments and strings, breaks on line wrapping, and
needs a maintained exclusion list -- more standing liability than the ~60-line
typed tool, against code-is-liability. lintstutter is therefore a sibling of
tools/lintfrom, not a make grep.

Scope discipline: this gate catches the specific stuttering-accessor case only.
It is deliberately NOT a general name/type-drift detector and does not chase
lowercase-local stutters (host.Host(), case differs from the exported method);
that softer, open-world signal is left to authoring discipline, not gated.

Regression proof: the acceptance is red-on-reintroduction, not green-on-HEAD. A
testdata package must carry a bad line (a value typed differently from its
accessor, read as the stutter) that the gate flags, and a good line (a value
whose type matches its name) it must pass; the tool's own test asserts the
non-zero exit comes from the bad line only.

Sequencing and dependency: the gate must gate an already-green tree, so it is
authored only after the Host-to-Address endpoint consolidation has landed. The
byte-exact instruction (tool source, make-target wiring, testdata fixture) is
authored ephemerally against the landing commit of that consolidation and
discarded after it lands; nothing byte-exact is stored here. No-exec does not
apply -- this is a build-time lint like lintfrom, outside the attested runtime
path.

Optional second arc: spec-layering, if the operator wants the gate bound to the
concept-tier work that surfaced it. Left single-arc type-cleanliness here, since
the gate is a standing typing invariant independent of the layering structure.
