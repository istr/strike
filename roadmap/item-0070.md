Discovery (at the time of writing): lane-run status is fragmented across three
homes -- lane.State (Outputs, Steps, Provenance; RWMutex-guarded), runState
(specHashes; imageFromRefs is removed by item-0069), and a loose networkRecords
map on the otherwise-config runContext. Every accumulator is keyed by step and
written by exactly one step (single-writer-per-key); cross-step readers only
read already-completed predecessors, guaranteed today by topological order.

Ratified direction: one structure in internal/lane (the layering home, M1), built
as one write-once record per step (the shape, M3). lane.State grows into that
structure; networkRecords folds in (internal/lane -> internal/capsule is an
allowed intra-services edge); the spec hash folds in as a bare digest value (no
registry import). Because each per-step record is written once and published at
step completion, the central RWMutex dissolves into per-record publication, which
is the lock-free basis for concurrent execution. lane.State and StepResult are
hand-written Go, not CUE-generated, so this is not a CUE-schema change; the
CUE-bound attestation predicate assembled in internal/deploy is unaffected as
long as it still receives provenance, output digests, and network records.

Execution is expected to span multiple PRs under one-concern-per-PR: the
behavior-neutral state consolidation and RWMutex removal land first; the
behavior-changing concurrent scheduler (replacing the sequential dag.Order loop)
lands separately.

Open forks to resolve with the operator at authoring time, not here:
- Spec-hash membership: fold the spec hash into the per-step record, or keep it as
  cmd/strike execution-only scratch since it is not attestation-facing.
- Outputs shape: keep a flat index keyed by output-ref, or nest outputs inside the
  per-step record, and the resulting Resolve path.
- Scheduler mechanism: worker pool versus per-step goroutine with an errgroup, and
  how predecessor happens-before and record publication are enforced.

Sequenced after item-0069 (which removes the imageFromRefs accumulator first).
