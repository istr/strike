Deferred follow-on from ADR-046 D9: machine-enforce the layer direction rather than rely on the file boundary plus review. The original base-/api-/wire- filename-prefix basis is dead -- the spec-layering reorg produced per-package tiers (primitive, concept, foundation/transport), not prefixed files within one package -- so the lint keys on the current package and tier layout instead. The Go side already enforces its layering via go-arch-lint; this is the mandatory CUE-side counterpart in the build gate. Whether the rule is a custom lint (tools/lintfrom mold) or an off-the-shelf CUE linter, and the exact classification predicate, are settled at authoring against the landed tier structure. Still assumes the internal API is wired end to end (item-0001).
