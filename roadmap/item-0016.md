Deferred follow-on from ADR-046 D9 (machine-enforced direction -- a separate package or a CUE lint -- is deferred; the file boundary is the first step). Not itemized in the legacy roadmap's implementation list -- surfaced during the roadmap-store coverage review. The file boundary (artifact-api.cue, D13) landed at 8de46bd; this item is the enforcement layer on top of it. Depends on item-0001 wiring the internal API end to end. Mechanism ratified: a custom CUE-direction lint (tools/lintfrom mold) keyed on the spec-layering base-/api-/wire- filename prefixes, forbidding an api-prefixed spec from naming a wire-only definition (base and api references stay allowed). Scheduled after the spec-layering reorg, because it classifies on the base/wire file separation that reorg produces. The package-split + go-arch-lint alternative was rejected as a reversal of ADR-047's single-package decision -- it belongs in an ADR-047 amendment, not here.
