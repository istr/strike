Ratified F1 = C-first. The attest-gengotypes spike found the signing path uses plain json.Marshal (internal/deploy/deploy.go), so JSON key order equals Go struct field order and is wire-significant; generating the attest DTOs would change field order (CUE source order vs fieldalignment) and break every reproducibility golden. Canonicalizing the signed payload makes the bytes order-independent, so the migration and all future reorders are wire-neutral -- the operator's call that relying on code order on the wire is too fragile on the way to beta. Security-sensitive (signing and verification path): needs a short design pass first -- which scheme (JCS vs sorted-keys), and how it interacts with DSSE PAE and the offline verifier's exact-byte re-check (DSSE carries the payload bytes, so canonical-on-produce is what makes reproduction order-independent). One deliberate golden regen when it lands. Head of the attest-migration sequence: lands before the attest cluster of item-0050; independent of the order-preserving field fixes item-0053 and item-0054.
