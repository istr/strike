Surfaced while wiring item-0001 (named-layer output model). The producer stamps
one OCI layer per output with the output id as its descriptor annotation; the
consumer selects by that id. Two outputs sharing an id within one step would
make layer selection ambiguous and overwrite the lane-state registration. Ids
are #Identifier; the outputs list is `[...#FileOutput]`, which does not
constrain id-uniqueness across elements on its own. Prefer a CUE expression of
list-element id-uniqueness; if that is not cleanly expressible, fold the check
into validateLane at DAG build (where step ids and output refs are already
validated). Not a basename concern: distinct ids with the same path basename
are fine, since each output is its own id-named layer.
