Three hand-maintained package enumerations exist and diverge by design: the gengotypes list in the generate target (the packages whose Go is generated, which includes output), the runtime roots built by internal/schema (attest, lane, provenance, crossval, trustlayers plus their import closure), and the embed list (the union the roots need, which excludes output). output is codegen-only and correctly not embedded; crossval and trustlayers are built from the embed for the conformance and vector tests. The drift is purely in the prose: the crossval comment predates the conformance-test rework that made internal/schema build it, and the README oversimplifies the embed as the whole tree. Single-sourcing the enumeration itself (a glob plus an embed-covers-the-roots check) is the structural follow-on and rides with the build-toolchain item (item-0035), not this docs change.
