The empty case is needed for no legitimate production state. The producer always computes laneDigest from the lane file bytes; the verifier sealed-predicate check treats an empty attestation laneDigest as an error (carries no laneDigest), so the verifier contract is already stricter than the schema and the schema is the loose side. The verify-side UC1 empty laneDigest is a separate object (the CLI policy comparison target, empty meaning do-not-pin), not the attestation field, and is unaffected. Blast-radius at execution: deploy-package test fixtures that set laneDigest empty as an unused placeholder. At least the SLSA-provenance validity test validates against the CUE schema and will need a real placeholder digest; the signing-rejection, unmarshal, and projection fixtures appear to be Go-logic only and need confirmation rather than change.
