Direct follow-on from the ADR-046 implementation retrospective. The output model
ended with three distinct keys sharing the token "Layer" at the symbol level --
LayerID (identity), OutputLayerName (in-layer content root), LayerDiffID (engine
selection key) -- one word for three levels. OutputLayerName carried both
confusing tokens: "Layer" (wrong level; it is a path inside the layer, not the
layer's id) and "Name" (the overloaded token corrected across several sessions).

Renames: OutputLayerName -> OutputContentPrefix (the re-rooting destination
prefix, i.e. the content root inside the layer); LayerID -> OutputID (the value
is out.ID, the output identity); annotation key dev.strike.output.layer ->
dev.strike.output.id. LayerDiffID stays -- diff_id is a genuine OCI-layer
property, so "Layer" is correct there; reserving "Layer" for that one real
layer-level key removes the overload.

The annotation-key change alters produced manifest bytes (the annotation sits on
the layer descriptor) but not consumer behavior: selection is by diff_id and the
annotation is advisory. Goldens that pin the annotation or the manifest digest
regenerate.

Sequencing. Land this before the spec-layering file reorg (items 0020-0025),
whose contract is "no name change" -- that reorg should move already-corrected
names, not chase renames across new file homes. Land it before the output-model
follow-ons (item-0026 disjointness, item-0016 direction enforcement) so those are
authored against the corrected vocabulary. Note: item-0026's notes still describe
annotation-based layer selection; the real consumer selects by diff_id -- align
that when item-0026 is authored. See the ADR-046 output-key vocabulary amendment.
