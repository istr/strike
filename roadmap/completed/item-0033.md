Drift-invariant notes:

Direction: the CUE package structure should mirror the Go package structure.
internal/transport is already its own Go package that internal/lane imports.
The CUE side does not mirror this: the transport vocabulary (#Host, #TLSTrust,
#FingerprintTrust, #CABundleTrust, #EngineConnection, #EngineUnix/TLS/MTLS)
lives in specs/base-transport.cue under package lane, @go(-) hand-written in
internal/transport, with the peer host and trust fields aliasing their Go type
via @go(...,type="...internal/transport".X). This item closes that asymmetry by
giving the transport vocabulary its own CUE package transport, so the type's
home is the same -- transport -- on both the CUE and the Go side.

Relationship to ADR-047 (requires an append-only amendment): ADR-047 decisions
(3) and (6) deliberately kept the generated layers (base/api/wire) in one CUE
package and the transport subset @go(-) hand-written in internal/transport. The
recorded reason is NOT that cross-package CUE imports are impossible -- the
attest package already imports specs:lane and names lane.#X qualified. The
recorded constraints are narrower: gengotypes maps one CUE package to exactly
one Go package, and adding an import statement to a CUE file makes that file's
@go() package target be ignored. attest is unaffected only because it is
hand-written, not gengotypes-generated. This item revisits the transport-in-lane
placement and so needs an ADR-047 amendment, authored at execution, never here.

Load-bearing open question -- a spike before any byte-exact instruction:
whether a gengotypes-generated consumer (specs/base-peer.cue, specs/wire-lane.cue)
can import the new transport package and still emit its own field @go() targets
correctly, given ADR-047 (3)'s "@go() ignored on import". The attest precedent
proves imports work only for a hand-written consumer; the generated-consumer
case is unverified, and no evidence was found that the gengotypes behaviour has
changed since ADR-047. Also to measure: how the transport CUE package avoids
emitting a Go type that collides with the hand-written internal/transport
(likely stays @go(-) / non-generated, or targets a distinct home). This is
generator-internal behaviour: measure in a throwaway worktree, do not guess. If
the generated-consumer import is not workable, this item does not land and
ADR-047 (3)/(6) stand as the recorded reason.

in the CUE-first state; run after it.
