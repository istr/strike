Drift-invariant notes:

Direction (ratified 2026-06-25): after item-0032 made the digest family
type-clean, the same invariant is extended to the whole internal Go API
surface. A type conversion sitting inside a call's argument list --
someCall(T(x), ...) -- is a defect: it means the callee or the source is
mistyped. The fix is to type the source or widen the callee, never to wrap the
call. The survey at the item-0032 landing found ~60 production and ~32 test
sites in the string(named) direction (a named-typed value cast down to a string
parameter), spanning Identifier, ImageRef, Host, the path types, Duration,
Base64, SecretSource, and TrustRootRef. None in the named(raw) direction remain
in production.

Three site classes and their ratified treatment:

1. Strike-internal callees taking string -- widen the parameter to the named
   type. The conversion then disappears at every call site. Examples:
   laneState.Register, registry.WrapDigestRef/WrapTag, state.RecordProvenance/
   CollectProvenance, dag.findOutput/addEdge/CollectPeers, deploy.signStatements/
   ingestRecords/resolveArtifactDigests/startUnitCapsule, captureKey,
   transport.New (Identifier); registry.FetchBaseSBOMReferrers, pullVerified,
   baseRepo, sha256Hex (ImageRef); transport.DialVerified, canonicalize,
   formatHost (Host); FetchTrustRoot (TrustRootRef).

2. Genuine foreign boundaries (stdlib, go-containerregistry, spdx, ssh, json) --
   the callee's signature is not strike's to change, so the conversion is
   irreducible. Fork A is ratified A1: the conversion is owned by a method on
   the named type (e.g. (RelPath).Dir, (AbsPath).HasPrefix, (Host).Canonical),
   which is semantically clearer at the call site -- it names what the
   conversion does, not merely its type -- and concentrates the boundary to one
   place per operation. The method body still hoists the conversion to a named
   local (s := string(p); ... path.Dir(s) ...) so the call-argument invariant
   holds inside the method too. Inline assignment hoists at the call site are a
   last resort, used only for genuine one-offs and justified in the commit body.

3. Composite-literal fields and map keys. Fork B is ratified: a map[string]X
   keyed by a named-type value (declaredPeers, dag.reverse, ...) becomes
   map[NamedType]X throughout, so lookups and inserts carry the type. Fork C is
   ratified per-field: a struct field receiving string(named) is retyped to the
   named type when the struct is strike-internal, and kept string when the field
   is a CUE-mirrored wire DTO; the call is decided at authoring against the
   then-current pin.

Out of scope: conversions into or out of CUE-mirrored wire DTOs and inbound
untrusted JSON, where string is the conformant boundary type (the same
boundaries item-0032 left as string: the libpod raw-decode struct,
BaseSBOMReferrer, verify.go's extracted laneDigest). These are documented
boundaries, not violations.

Sequencing: per type family, so each family's parameter/field/key widening
cascades coherently and gates on a family-scoped conversion grep. Recommended
order by size: Identifier (largest -- dag, state, deploy, cmd), then Host,
ImageRef, the path types (AbsPath/RelPath, which carry the A1 method set), then
the remainder (Duration, Base64, SecretSource, TrustRootRef). The final
instruction extends the tree-wide f(T(x)) gate to all named types and amends
CODE-STYLE.

Dependency: builds on item-0032. Each instruction anchors on the landing commit
of the prior one (the digest family must already be type-clean, or the tree-wide
gate cannot pass). The byte-exact instructions are authored ephemerally against
the then-current pin; nothing byte-exact is stored here on purpose. The
CODE-STYLE amendment is an append-only edit made at execution time, never here.

## Final summary

Type-cleanliness arc concluded and structurally enforced. tools/linttypeconv (wired into make check) fails the build on any strike named-type conversion sitting in a call argument, exempting a type's own package (and its external test package) and one central allowlist entry (mediator peer-host canonicalization, deferred until the resolver path is typed). The whole tree -- production and tests -- is green: remaining cross-package call-argument conversions hoisted to named locals or owned by typed constructors; lane.Ptr[T any] eliminated in favor of typed primitive pointer/value constructors; ImageConfig.user promoted to a named primitive.UserSpec (constraint-free, schema unchanged). CODE-STYLE carries the type-conversion-ownership rule and its permitted forms.
