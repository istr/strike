The clean native-DAG alternative to the chosen Option 2 (one generated CUE
package with file-prefix layering). Recorded as the documented fallback, not
scheduled.

Why deferred: it reverts two ratified decisions -- flat specs/ (no
subdirectories) and no new Go packages -- and carries a large Go blast radius,
since every consumer of the generated lane types would split across the new
package homes.

Why it is the only path to a native DAG: the gengotypes-collapse spike found
that cue exp gengotypes (v0.16.0) cannot fold several CUE packages into one Go
package -- an import statement makes the @go(pkgname) override be ignored, the
CUE-to-Go mapping is 1:1, and it refuses two Go packages in one directory. So
separate CUE packages in subdirectories generating separate Go packages is the
only way to get a CUE-native and Go-side (go-arch-lint) layering DAG, including
attest depending on base alone.

Reconsider only if layering purity is later weighted above the churn and the
two reverted decisions. See ADR-047 and docs/SPEC-PACKAGE-LAYERING.md.

## Final summary

Option 2 was ratified and implemented via ADR-047; the contract/ refactor landed with the chosen approach
