Residual fallout of the specs/ -> contract/ package rename (item-0037,
ADR-047): the directory, Makefile, and embed were updated, but prose and
comment references to the old specs/ path were left behind across docs and a
few Go files. contract/README.md is the worst offender -- its schema table
still lists specs/*.cue paths. This is a docs-and-comments hygiene pass, not a
code change.

Excluded by construction: ADRs (append-only; their specs/ mentions are correct
for their decision-time context) and the roadmap archive (historical). Genuine
narrative references to the rename itself (e.g. in SPEC-PACKAGE-LAYERING.md)
stay. The acceptance search must therefore be scoped to live references, not a
blanket no-match.
