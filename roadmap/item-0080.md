Register rows R-01, R-08, R-09, R-11, R-17, R-30, R-31. Forks A and B are ratified (both option 1). Fork A: the unit key is a comparable value struct CapsuleUnit{Step primitive.Identifier; Capture primitive.Identifier}, empty Capture meaning "the step itself", used directly as the comparable map key, as the constructor parameter, and rendered for display/logs by one owned method; both captureKey twins and the "capture:<step>:<capture>" string protocol are removed and stepPorts becomes map[CapsuleUnit]HostPorts. It is accepted as in-process identity / disjunction glue, a hand-written comparable struct, not a CUE schema gap. Fork B: the canonical host is endpoint.CanonicalHost (hand-written, concept tier) with the single constructor Canonicalize(primitive.Host) (CanonicalHost, error); the mediator peer map, resolver allowlist, and ephemeral-CA cache key on it, while QueryRecord.QName stays string by design (untrusted wire name from container DNS packets); the linttypeconv M3 allowlist deferral is retired with this item as its owner. This is the highest-judgment item of the program; the capture-composite domain fact (retrospective 4.1.1) is the reason the unit key is a ratified design, not an executor discovery.
