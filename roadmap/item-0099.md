The engine and carriage discriminators established the pattern: a named
top-level string disjunction, projected onto each literal-pinned arm via the Go
type override, so the enum generator emits exported constants a cross-package
consumer can name, and consumer switches use those constants so the exhaustive
linter checks them. Three struct-arm unions still restate their discriminator
per arm: the trust anchor (certFingerprint, caBundle) in endpoint, the deploy
method (kubernetes, registry, custom) in lane, and the provenance source (git,
tarball, oci, url) in provenance. Apply the same pattern to each; the arm
accessor methods project the typed field back to string at the interface
boundary, and the marshal and unmarshal seams the owning package holds are where
the wire literal legitimately stays.

One caution carries over from the engine and carriage work. Typing a
switched-on field pulls every switch and comparison over it into the exhaustive
ripple, and the exhaustive linter matches by named member rather than by value:
a bare literal case does not count as covering the member it equals. So every
literal-cased switch over each retyped field must convert to the named constants
or carry a default, or the linter reports all members missing. Measure each
union's consumer set across the whole tree, not only the owning package.
