Defect surfaced by item-0075. #Sealed.resolver is a required, non-optional
#ResolverRecord (unlike engine? and observedPeers?), so a deploy with no DoT
resolver (e.g. a registry deploy) serializes an all-empty resolver object and
relies on bare string accepting "" for host and serverCertFingerprint.
Tightening those two fields to endpoint.#Authority / primitive.#Digest rejects
the empty case, breaking TestValidateAttestation_EmptyDigestsAllowed. item-0075
therefore left these two fields as string and routed the typing here, where the
optionality can be modeled rather than papered over.

The intended fix is to make the resolver block optional (resolver?:) with an
absent-when-no-resolver Go representation, so the constrained field types apply
only when a resolver was actually probed. This is a wire change (empty object
-> omitted), hence the golden regeneration, and it updates the empty-digests
test to assert absence instead of empty-string validity.

Rejected alternative: host: #Authority | "" and serverCertFingerprint: #Digest
| "". It keeps bytes identical but (a) introduces an inline empty-disjunction in
contract/ right where item-0090 is removing inline disjunctions, and (b) is
generator-uncertain -- whether cue exp gengotypes emits a typed Go field or
collapses "X | \"\"" to string must be measured, not assumed.

Open question: whether to fold this into item-0049 (resolver decl restructure)
or keep it as the attestation-record counterpart. They touch different objects
(observed record vs input decl) and can land independently; sequencing is the
operator's call.
