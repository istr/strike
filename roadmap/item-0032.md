Establish CUE-first typing for scalar value constraints (direction ratified
2026-06-24). Today the specs carry a wild mix: regex-constraint scalars are
variously generated named types (ImageRef, Duration, AbsPath, RelPath,
ArtifactType, FileArtifactType, SecretSource), dead generated types every
field overrides to plain string (Identifier, Path), plain-string atoms added
in item-0024 (Sha256, Base64, GitCommit, all @go(-) plus field-level
type=string), and one hand-written struct hidden behind @go(-) (Digest). The
@go(-) annotation has been doing two unrelated jobs -- legitimately hiding
disjunction sum types, and illegitimately hiding a hand-written internal API
type whose stored form (algorithm + hex) differs from its wire form
("sha256:<hex>"). The second use is the structural defect: a wire/internal
collapse under one name, papered over by suppressing codegen.

Ratified resolution:

- Regex-constraint scalars are the type. They carry no @go(-) and no
  field-level type=string; the constraint defines a named Go string type that
  fields use directly (id: #Identifier, laneId: lane.#Identifier). @go(-) is
  reserved exclusively for disjunctions (the Peer, provenance-record,
  DeployMethod, ArtifactSource, TLSTrust, EngineConnection unions and their
  hand-modelled variant structs).

- The shared 64-hex body is single-sourced as a hidden CUE fragment
  (e.g. _sha256Hex: "[a-f0-9]{64}") composed by interpolation into the
  anchored patterns: #Sha256 =~ "^\(_sha256Hex)$", #Digest =~
  "^sha256:\(_sha256Hex)$", and optionally #ImageRef and the 64-hex branch of
  #GitCommit. This is composition by shared fragment, not & refinement
  (a prefixed digest is not a subset of the bare hex, unlike AbsPath/RelPath
  which are subsets of #Path).

- The digest concern splits by layer. The wire form stays in base-/lane- as a
  generated string type (type Digest string), serialising naturally. The
  structured internal contract moves to api-digest.cue as its own generated
  type (e.g. #DigestRef -> type DigestRef struct{Algorithm, Hex string}) with
  a conversion bridge at the boundary. Neither carries @go(-).

- Hand validation is preserved by amending, not overriding. gengotypes emits
  only type declarations and JSON tags, never methods, so a hand-written
  sibling file in the same package adds ParseDigest, accessors, the algorithm
  allow-list, and any Marshal/Unmarshal needed -- on the generated type, with
  no @go(-). digest_type.go loses its type declaration and keeps its
  behaviour.

First step is a sizing spike, not a byte-exact instruction: in a throwaway
worktree, strip @go(-) from the regex scalars and the field-level type=string,
apply the digest wire/internal split, run make generate and make check, and
report (a) the generated-types delta, (b) the consumer compile-break set --
every Go site that reads these fields as plain string or reads
Digest.Algorithm/.Hex, and (c) confirmation against the real gengotypes output
that it emits no methods (so amend-not-override holds). That measurement
decides whether this lands as one instruction or a multi-instruction arc and
bounds the churn before any contract bytes are written.

Ground the arc against d638631b (post item-0024). The rule, once landed, is
codified in docs/CODE-STYLE.md: regex constraint -> named type, no annotation;
@go(-) -> disjunctions only; wire and internal contracts are separate types in
separate layers, never one type behind a suppressed codegen.
