The last two shell-dependent Makefile gates. lint-ascii is a grep over source
files; lint-adr-index is a shell loop that ADR-INDEX.md documents as a relied-on
CI guard, so its go-native replacement must keep that guarantee and the
ADR-INDEX.md reference must be repointed. Retiring these is doubly motivated:
they block Makefile elimination (item-0096) and they are the build-time echo of
the README first-line "no shell" posture (build-time, not the attested runtime
path, but consistent). Independent of item-0094; both feed item-0096.
