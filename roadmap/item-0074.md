Closes mechanisms M4 and M5 from the retrospective: fresh contract fields regress where no gate observes, and lossy annotation redirects decay silently. The named-disjunction check is the standing gate for ADR-049 rule (5): item-0090 hoists the five present violations once, and this gate keeps a sixth from being authored -- an inline disjunction is the same fresh-field regression as an unreused primitive, so it belongs to the same M4 tool rather than a new one. Whether the three checks live as modes of the existing CUE-coverage linter or as a separate tool is an implementation choice inside the item; the disjunction check needs the parsed CUE the coverage linter already loads, which argues for a mode.
