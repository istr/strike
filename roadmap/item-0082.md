Register rows R-26, R-27, R-28, R-45. Fork H is ratified (option 1): image-ref entry is ParseImageRef(string), which still rejects the empty value (it does not adopt ParseDigest's empty-round-trips-absence contract, since item-0083 retires exactly that). The one production site that can supply an empty value (cmd/strike/verify.go, the trust-root-ref flag) guards on absence before parsing, which is where item-0083 finds the sentinel. The executor substring re-check (strings.Contains(img, "@sha256:")) and the detyping that feeds it are deleted, but the ADR-045 refusal is retained, re-expressed as a typed opts.Image.Digest() check with execute_digestguard_test.go kept: a named string type cannot be made valid by construction in Go, so a validating entry makes the last structural check before an engine call typed and owned, not redundant. resolveDigest's unreachable non-digest fallback and its *registry.Client parameter are removed. The ingest-parse and hex-constructor changes are unblocked.

R-40 is delivered and dropped from this item. The artifact-type vocabulary now
comes from the primitive package: the file-artifact-type definition generates
its value constants, a method on the type owns the comparison, and the untyped
constants in the command package are gone (commit c40ebe9). The orphaned
three-value artifact-type definition was removed in the same change, since no
CUE field and no Go type referenced it.
