The collisions are name clashes between distinct concepts, not redundancy. The
method is the deploy mechanism (kubernetes, registry; a real discriminator) and
the target is the deploy destination identity used to pair pre- and post-state
digests across deploys, whose type is a free string.

ADR-051 D6 settles the merge-or-distinct half: the attestation target identity
is retained and is explicitly not the transport destination, and the push
destination is a separate mutable reference. The fields stay distinct. Three
questions are left. Both the deploy spec's attestation target identity and the
registry method's transport destination are named target, so they need distinct
names. The target type is a free string beside the method type enum, whose
membership ADR-051 D5 changes by removing custom. And namespace is declared on
both the kubernetes method (required) and the target (optional), so which one
is authoritative when the method is kubernetes is still open; it is settled
first. Resolve by a small reasoning or spike task, not from the cuff. A
follow-on fix item applies the ratified decision.
