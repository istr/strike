Reworks the resolver-shape decision item-0042 settled. item-0042 homed the
resolver on endpoint.#TLS (the packed-authority #Authority address) and kept
the IP-literal narrowing as a Go boundary check rather than a CUE type. This
item reverses both: the resolver gets its own endpoint shape with a structured
address -- a bare IP literal beside a separate port -- and the IP-literal
constraint is promoted to a single-sourced primitive type composed with the
ADR-047 loose-body-plus-interpolation pattern (the same shape _sha256Hex feeds
#GitCommit). Rationale (drift-invariant): the resolver is strike's only
IP-addressed peer, so it is the one place a packed authority cannot safely carry
the address -- an IPv6 literal's colons collide with the host:port separator.
Keeping ip and port as sibling fields is the structural fix and is the exact
reason #Authority cannot carry the resolver address.

Scope boundary (operator): lane wire format only, not attestation. The observed
sealed record (#ResolverRecord) is a separate flat host:string field recording
the probed identity; it is deliberately decoupled from the declared lane shape
and is not retyped here. The trust-layers classification keys the resolver by
name, not by wire shape, and is unaffected.

Coordination -- ratified item-0046: item-0046 (ratified, currently first in the
execution order) ties the observed sealed record to "the declared endpoint.TLS".
Moving the resolver onto its own endpoint shape changes that linkage target.
Open sequencing question for the operator: land this rework before item-0046 (so
0046 links against the settled resolver shape) or after (and re-point 0046's
open question from endpoint.TLS to the resolver shape). The wire rename itself is
attestation-independent, but the lane bytes feed laneDigest, so any sealed-golden
delta is a delegate-measure at execution time, not an assumption.

Coordination -- dns-resolver arc: the optional port defaulting to 853 lands
item-0018's goal structurally, so item-0018's packed-host framing ("1.1.1.1"
equals "1.1.1.1:853") becomes obsolete; item-0018 should be retired or re-scoped
against this shape. item-0017 (combined IP plus hostname with SAN/CN
verification) layers a verification anchor on top of this structured shape and
should be authored against it, not the pre-rework #TLS form; the ip/port split
leaves room for a later hostname sibling without disturbing it.

CUE/Go pipeline: the schema change spans primitive (scalars) plus endpoint plus
lane, so the order is schema -> make generate -> verify generated types -> touch
Go callers. Golden-affecting potential via the sealed laneDigest path: measure,
do not assume.
