The capstone of the build-toolchain arc: the item that actually deletes the
root Makefile. It is the LAST build-toolchain item before the tag and is a hard
downstream of every other gate going go-native -- item-0035 (cue codegen/fmt),
item-0093 (Go-type linters), item-0094 (cuelint), item-0095 (shell gates) --
plus item-0063 resolved to option B, whose dogfood strike lane is the
orchestration that replaces the aggregate `make check`. Until that lane exists
there is nothing to delete the aggregate into. The residual thin wrappers
(build flags, golden-update, test, integration, vuln, lint-arch, lint-ci) move
to go:generate directives, the gate lane, or documented go commands. The docs
migration is in scope, not a follow-on: leaving `make X` in AGENTS.md and the
workflow docs after the file is gone is a broken instruction, not cosmetic.
Note item-0097 retires the second (sigstore-local) Makefile; both must land for
a tree with zero Makefiles.
